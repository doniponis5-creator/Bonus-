# ЗАДАЧА: Создать расширение для 1С:УТ 11.5 — полная интеграция с SBonus+

У меня есть система лояльности SBonus+ (FastAPI бэкенд) для магазина **Смарт Центр** (Кыргызстан, валюта KGS/сом). Нужно создать одно расширение для 1С:УТ 11.5 которое делает ВСЁ нижеперечисленное.

**Версия 1С:** Предприятие 8.3 (8.3.27.1719), Управление торговлей 11.5.7.308

---

## ⛔ КРИТИЧЕСКИ ВАЖНЫЕ ПРАВИЛА БЕЗОПАСНОСТИ

**Ты получил доступ к РАБОЧЕЙ базе 1С магазина. Это ПРОДАКШН — здесь реальные данные, реальные клиенты, реальные продажи. Одна ошибка = магазин встанет.**

### КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО:
1. **НЕ МЕНЯТЬ** существующие объекты конфигурации (документы, справочники, регистры) — ТОЛЬКО ЧИТАТЬ
2. **НЕ УДАЛЯТЬ** никакие данные, записи, документы, элементы справочников
3. **НЕ РЕДАКТИРОВАТЬ** существующие модули, обработки, отчёты основной конфигурации
4. **НЕ ВЫПОЛНЯТЬ** INSERT, UPDATE, DELETE напрямую в базу — только SELECT/ВЫБРАТЬ запросы для чтения
5. **НЕ ПЕРЕЗАПИСЫВАТЬ** регламентные задания, подписки, роли основной конфигурации
6. **НЕ ТРОГАТЬ** учётные записи пользователей, права доступа, настройки безопасности
7. **НЕ ОТКЛЮЧАТЬ** существующие механизмы (обмены, синхронизации, регламентные задания)

### РАЗРЕШЕНО:
1. **ЧИТАТЬ** данные запросами (ВЫБРАТЬ / SELECT) — для диагностики
2. **ЧИТАТЬ** метаданные (Метаданные.Документы, Метаданные.Справочники и т.д.)
3. **СОЗДАВАТЬ** только в РАСШИРЕНИИ (новые модули, константы, рег. задания, подписки — всё с префиксом SB_)
4. **ПИСАТЬ** код только в расширении SBonusPlus — ни одной строки в основной конфигурации
5. **ТЕСТИРОВАТЬ** только через тестовые функции расширения (ТестСоединения, ТестОтправкиТовара)

### ПРИНЦИП РАБОТЫ:
- Расширение = **надстройка**. Оно читает данные из основной базы и отправляет в SBonus+ API. Оно НИЧЕГО не меняет в основной базе.
- При ошибке отправки — НИКОГДА не блокировать проведение документов. Используй Попытка/Исключение с записью в журнал регистрации.
- Все новые объекты ТОЛЬКО с префиксом `SB_` чтобы не конфликтовать с основной конфигурацией.

## ШАГ 2: ПОСЛЕ ДИАГНОСТИКИ — НАПИШИ РАСШИРЕНИЕ

На основе результатов диагностики напиши полное расширение 1С (общий модуль + константы + рег. задания + подписки).

### Расширение должно делать ВСЁ из этого списка:

#### А. ТОВАРЫ И ОСТАТКИ
1. **Синхронизация товаров** (рег. задание, ежедневно 06:00) — отправить весь каталог: SKU, название, категория, цена, себестоимость, остаток, штрихкод, единица, поставщик
2. **Обновление остатков** (рег. задание, каждый час) — только SKU + текущий остаток
3. Пакетная отправка: товары по 500, остатки по 1000

#### Б. ЧЕКИ И БОНУСЫ
4. **Отправка чека при продаже** (подписка на ЧекККМ.ОбработкаПроведения) — телефон клиента, сумма, позиции (SKU, кол-во, цена). API сам начислит бонусы
5. **Проверка баланса бонусов** — кассир вводит телефон, GET запрос к API, показать баланс
6. **Списание бонусов** — кассир указывает сумму списания, POST в API, скидка в чеке
7. **Возврат** — при возврате товара отправить данные в API для отмены бонусов

#### В. КЛИЕНТЫ
8. **Регистрация клиента из 1С** — если новый покупатель, отправить ФИО + телефон в API
9. **Поиск телефона** — из контрагента, из дисконтной карты, из контактной информации

#### Г. ДОЛГИ И РАССРОЧКИ
10. **Синхронизация долгов** (рег. задание, каждые 2 часа) — по каждому контрагенту: сумма долга, номер договора, дата договора
11. Если есть документы рассрочки/графики платежей — отправить их тоже

#### Д. БЕЗОПАСНОСТЬ И СЕРВИС
12. **HMAC-SHA256 подпись** — каждый запрос подписан, заголовок X-Signature
13. **Логирование** — все действия в журнал регистрации 1С
14. **Тест соединения** — проверить связь с API
15. **Тест отправки товара** — отправить один тестовый товар
16. **Константы:** SB_APIUrl, SB_WebhookSecret, SB_IntegrationEnabled
