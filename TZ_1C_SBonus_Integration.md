# ТЗ: Интеграция 1С с S Bonus+ (Смарт Центр)

## Общее описание

Нужно настроить автоматическую отправку данных из 1С в систему лояльности S Bonus+.
Сервер принимает HTTP POST/GET запросы в формате JSON.
Base URL: https://api.smartcentr.store

Все запросы — Content-Type: application/json.

---

## ВАЖНО: Архитектура — очередь (не прямые вызовы!)

НЕ отправлять HTTP запросы напрямую при проведении документа — это замедлит 1С.

Правильная схема:

1. При проведении документа → записать данные в РегистрСведений (например SBonusQueue)
2. Фоновое задание (каждые 15-30 секунд) → берёт записи из очереди → отправляет HTTP → при успехе (201) удаляет из очереди
3. При ошибке сети → retry_count + 1, попробует снова через 30 сек
4. После 5 неудачных попыток → пометить как "ошибка", записать в лог, пропустить

Поля регистра очереди:
- id (номер записи)
- event_type (purchase / spend / refund / register / debt_update / expense)
- payload (строка JSON — тело запроса)
- status (pending / sent / error)
- retry_count (число)
- created_at (дата и время)
- sent_at (дата и время отправки)
- error_text (текст ошибки если была)

## Эндпоинт 1: Покупка (начисление бонуса)

Когда: при каждой продаже (проведение чека/реализации)

POST /api/v1/webhook/1c/purchase

```json
{
  "customer_phone": "+996557100505",
  "purchase_amount": 15000.00,
  "branch_id": "uuid-филиала-из-SBonus",
  "cashier_id": "uuid-кассира-из-SBonus",
  "receipt_number": "CHK-2026-001234",
  "items": [
    {"sku": "ART-001", "quantity": 2, "price": 5000.00},
    {"sku": "ART-002", "quantity": 1, "price": 5000.00}
  ]
}
```

Поля:
- customer_phone (обязательно) — телефон покупателя, формат +996XXXXXXXXX
- purchase_amount (обязательно) — общая сумма чека в сомах
- branch_id (обязательно) — UUID филиала (получите из админки S Bonus)
- cashier_id (необязательно) — UUID кассира
- receipt_number (обязательно) — номер чека, УНИКАЛЬНЫЙ (защита от дублей)
- items (обязательно!) — массив позиций чека:
  - sku — артикул товара из 1С
  - quantity — количество
  - price — цена за единицу

ВАЖНО: items обязателен! Без него не работает: P&L отчёт, товарная аналитика, ABC-анализ, остатки.

Ответ при успехе: 201
Ответ при дубле receipt_number: 409 (просто пропустить)

## Эндпоинт 2: Списание бонусов

Когда: клиент оплачивает часть покупки бонусами

Шаг 1 — проверка доступной суммы:
GET /api/v1/webhook/1c/check-spend/+996557100505?purchase_amount=15000

Ответ:
```json
{
  "balance": 1500.00,
  "purchase_amount": 15000,
  "max_spend_amount": 4500.00,
  "available_spend": 1500.00
}
```

available_spend — это максимум сколько можно списать (учитывает баланс и лимит 30%).

Шаг 2 — списание:
POST /api/v1/webhook/1c/spend

```json
{
  "customer_phone": "+996557100505",
  "spend_amount": 1500.00,
  "purchase_amount": 15000.00,
  "branch_id": "uuid-филиала",
  "cashier_id": "uuid-кассира",
  "receipt_number": "CHK-2026-001234"
}
```

Ограничение: максимум 30% от суммы покупки. Система сама ограничит.

## Эндпоинт 6: Долги / Рассрочки

Когда: при выдаче рассрочки, при каждой оплате рассрочки, при изменении статуса

POST /api/v1/webhook/1c/debt-update

```json
{
  "phone": "+996557100505",
  "full_name": "Абдумалик Шагирт",
  "amount": 50000.00,
  "total_amount": 120000.00,
  "paid_amount": 70000.00,
  "overdue_days": 15,
  "reference": "DOC-1C-2026-0042",
  "note": "Рассрочка на телевизор",
  "schedule": [
    {"date": "2026-04-01", "amount": 20000, "status": "paid"},
    {"date": "2026-05-01", "amount": 20000, "status": "overdue"},
    {"date": "2026-06-01", "amount": 20000, "status": "pending"}
  ],
  "payments_history": [
    {"date": "2026-03-15", "amount": 50000, "document": "PAY-001", "overdue_days": 0},
    {"date": "2026-04-20", "amount": 20000, "document": "PAY-002", "overdue_days": 5}
  ],
  "next_payment": {
    "date": "2026-06-01",
    "amount": 20000
  }
}
```

Поля:
- phone (обязательно) — телефон клиента
- amount (обязательно) — ОСТАТОК долга. Если 0 — значит полностью оплачен
- total_amount — общая сумма рассрочки
- paid_amount — сколько уже оплачено
- overdue_days — количество дней просрочки (0 = вовремя)
- reference (обязательно) — номер документа 1С. ЭТО КЛЮЧ: повторный запрос с тем же reference обновит запись, а не создаст новую
- schedule — график платежей (массив)
- payments_history — история оплат (массив)
- next_payment — следующий платёж

ВАЖНО: overdue_days влияет на кредитный рейтинг клиента (0-100 баллов). Система автоматически категоризирует клиентов:
- Чёрный список (90+ дней просрочки или 2+ долга с просрочкой >30 дней)
- Проблемный (30-90 дней)
- Мониторинг (1-30 дней)
- Надёжный (без просрочек)
- Новый (нет истории)

Кассиры видят этот рейтинг перед выдачей рассрочки.

## Эндпоинт 7: Синхронизация товаров

Когда: ежедневно (рекомендуется ночью, 02:00-04:00) или при изменении каталога.
Можно отправлять до 5000 товаров за раз.

POST /api/v1/webhook/1c/products-sync

```json
{
  "products": [
    {
      "sku": "ART-001",
      "name": "Samsung Galaxy A54",
      "category": "Смартфоны",
      "barcode": "4607123456789",
      "unit": "шт",
      "price": 35000.00,
      "cost_price": 28000.00,
      "current_stock": 15,
      "min_stock_level": 3,
      "supplier": "Samsung KG"
    },
    {
      "sku": "ART-002",
      "name": "Чехол для Samsung A54",
      "category": "Аксессуары",
      "barcode": "4607123456790",
      "unit": "шт",
      "price": 500.00,
      "cost_price": 200.00,
      "current_stock": 50,
      "min_stock_level": 10,
      "supplier": "Aksessuar KG"
    }
  ]
}
```

Поля товара:
- sku (обязательно) — артикул. ЭТО КЛЮЧ: если товар с таким sku есть — обновится, если нет — создастся
- name (обязательно) — название
- category — категория (Смартфоны, Аксессуары, Бытовая техника...)
- barcode — штрих-код
- price (обязательно) — розничная цена
- cost_price (ОБЯЗАТЕЛЬНО для P&L!) — себестоимость / закупочная цена
- current_stock (обязательно) — текущий остаток
- min_stock_level — минимальный остаток для оповещения (по умолчанию 5)
- supplier — поставщик

КРИТИЧНО: cost_price обязателен! Без него система не может считать валовую прибыль.
Формула: Валовая прибыль = Выручка (цена продажи) - Себестоимость (cost_price)

## Эндпоинт 9: Расходы магазина

Когда: ежемесячно (или при каждом расходе)

POST /api/v1/webhook/1c/expenses

```json
{
  "expenses": [
    {
      "category": "Аренда",
      "amount": 50000.00,
      "month": "2026-05",
      "description": "Аренда магазина за май",
      "reference": "EXP-2026-05-001"
    },
    {
      "category": "Зарплата",
      "amount": 180000.00,
      "month": "2026-05",
      "description": "ЗП сотрудников май",
      "reference": "EXP-2026-05-002"
    },
    {
      "category": "Доставка",
      "amount": 15000.00,
      "month": "2026-05",
      "description": "Доставка товаров",
      "reference": "EXP-2026-05-003"
    }
  ]
}
```

- category — любое название категории (свободный текст)
- amount — сумма расхода в сомах
- month — месяц в формате YYYY-MM
- description — описание
- reference — номер документа 1С (защита от дублей)

## Коды ответов

- 201 — Успех, данные приняты
- 400 — Неверный формат данных
- 403 — Webhook отключён в настройках или IP заблокирован
- 404 — Клиент не найден по телефону
- 409 — Дубль (receipt_number уже обработан) — просто пропустить
- 422 — Ошибка валидации полей
- 500 — Ошибка сервера — повторить позже
