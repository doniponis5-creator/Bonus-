Бэкенд
🐍
FastAPI (Python 3.11+)
Самый быстрый Python фреймворк. Автоматическая Swagger документация. Async/await — идеально для мульти-филиалов. Отлично подходит для Green API webhook. Мощная валидация через Pydantic v2.
fastapipydantic v2sqlalchemy 2.0alembic
🐘
PostgreSQL 15+
Надёжная реляционная БД. ACID транзакции — критично для бонусных операций. UUID primary key. Изоляция схем для мульти-филиалов.
postgres 15uuid-ossppg_cron
⚡
Redis 7+
OTP коды (с TTL). Кеш сессий. Rate limiting. Celery очередь задач. Быстрый кеш баланса. Чёрный список токенов.
redis 7celeryotp ttl
🔑
JWT RS256 + OAuth2
Access/refresh токены для админов и кассиров. Контроль доступа по ролям (RBAC). Разрешения по филиалам.
jwt rs256rbacrefresh rotation

💳 Списание бонуса

→
Максимум: 30% от суммы покупки

→

→

Клиент А пригласил друга
→
После регистрации друга: +100 KGS для А
Новый друг сделал первую покупку
→
Новому клиенту +50 KGS приветственный бонус

Стратегия повторных попыток (Retry)
🔄 Retry-логика для уведомлений и вебхуков
WhatsApp/SMS отправка не удалась
→
Повтор через 30с, 2мин, 10мин (макс. 3 попытки)
Все 3 попытки не удались
→
Статус: failed, уведомление в админ-панель
1С webhook таймаут (5с)
→
Ответ 202 Accepted, обработка в фоне через Celery
Ошибка БД при бонусной операции
→
Полный откат транзакции (ROLLBACK), ответ 500
Уровни безопасности
🔑
JWT RS256 + Ротация Refresh Token
Access токен: 15 минут. Refresh токен: 30 дней. При каждом обновлении refresh токен меняется. Redis чёрный список — после выхода токен не работает.
ACCESS_TOKEN_EXPIRE = 15 # минут REFRESH_TOKEN_EXPIRE = 30 # дней ALGORITHM = "RS256"
🛡
RBAC — Контроль доступа по ролям
Кассир видит только свой филиал. Админ филиала управляет только своим филиалом. Супер админ — видит и управляет всем.
super_admin → ["*"] branch_admin → ["branch:{id}:*"] cashier → ["customers:read", "bonus:earn", "bonus:spend"]
⏱
Rate Limiting — Защита от перебора
Логин: 5 попыток / 15 минут. OTP: 3 попытки / 10 минут. API общий: 100 запросов / минута. Реализовано через Redis счётчики.
LOGIN_LIMIT = "5/15min" OTP_LIMIT = "3/10min" API_LIMIT = "100/min"
📝
Неизменяемый журнал аудита
Каждая бонусная операция записывается в таблицу transactions. Триггер БД не позволяет удалить или изменить. ID кассира, филиал, время, сумма — всё сохраняется.
🔒
Безопасность вебхуков (1С и Green API)
1С webhook — белый список IP + подпись HMAC-SHA256. Green API webhook — проверка instanceId. Ключ идемпотентности — одна покупка не получит бонус дважды.
X-Signature: HMAC-SHA256(secret, body) X-Idempotency-Key: receipt_number
🌐
HTTPS + CORS + Валидация ввода
Весь трафик через SSL. CORS — только разрешённые домены. Pydantic v2 валидирует все входные данные. SQLAlchemy ORM — нет SQL инъекций. XSS санитизация.
💾
Резервное копирование и восстановление
Автоматический pg_dump каждые 6 часов. Хранение последних 7 дней бэкапов. Тестовое восстановление раз в неделю. Резервные копии шифруются перед выгрузкой.
BACKUP_INTERVAL = "6h" BACKUP_RETENTION = "7d" BACKUP_ENCRYPT = "AES-256-GCM"
